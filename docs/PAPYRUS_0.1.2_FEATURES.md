# Papyrus 0.1.2 功能说明

Papyrus 是面向生活秘书、文学写作、文科研究和文字工作的本地优先 AI 工作台。0.1.2 重点补强模型治理、秘书模式、多 Agent 蜂巢编排、写作辅助、长期记忆分层和工程可观测性。

## 关键特性

### 秘书模式

- 默认工作入口是秘书模式，由“秘书长”理解任务、规划路径、调用工作室 Agent，并把需要写入的正文以文稿补丁形式落到编辑器。
- 支持 `/plan` 先生成可协商规划，用户确认后再执行。
- 支持 `/goal` 长程目标模式，适合长篇小说、连续章节、研究报告和复杂运营任务。
- AI 工作中可以继续输入，内容会进入队列；用户也可以把队列项作为“引导”注入当前任务。

### 主站模型网关与 Auto 模型调度

- 内置模型不再固定显示旧模型名，而是在启动时从主站模型网关读取可用模型、上下文窗口和可用状态。
- 模型选择器使用固定弹层，避免被写作模式或底部输入栏裁剪。
- Auto 推荐模式会让秘书长按任务阶段选择模型：规划、分类、压缩、裁判优先轻量模型；正文生成、复杂推理和 ultra+hive 聚合优先高质量或大上下文模型。
- 常规设置中可限制 Auto 可调用模型，并查看 T1/T2/T3 分层和权重。

### 思考强度与 ultra+hive

- 思考强度固定为 `low / medium / high / ultra+hive`。
- `ultra+hive` 是最大思考强度加蜂巢 Agent 集群，适合长文、研究、合规、跨文档、复杂运营和 `/goal`。
- Hive 运行会显示计划 Agent、活跃 Agent、完成、跳过、失败、traceId、重试、断路器和超时状态。

### Hive 稳定性与可观测性

- 每次 Hive run 生成 traceId，便于追踪链路。
- 全局超时防止复杂任务无止境运行。
- 单 Agent 超时、最多两次退火重试和断路器机制降低失败成本。
- “共享黑板”实时显示路由、启动、完成、重试、超时、断路器和汇总事件。
- 早停机制会在高置信或连续无新增信息时停止重复协作，控制 token 浪费。

### 长期记忆与 towrite.md

- 个人记忆、项目记忆、towrite.md、写作知识和 story memory 统一作为长期上下文来源。
- 设置中明确说明长期记忆默认本地保存，不用于收集或上传用户信息，只用于增强 AI 对用户习惯和项目上下文的理解。
- 0.1.2 进一步把短期工作现场和长期记忆在代码与持久化层面分开：执行轨迹、todo、运行队列、临时目标和文稿补丁不再作为长期记忆持久化。

### 写作模式工具栏

- 选中文本后可使用降噪功能，先预览红绿改动，再确认写入。
- 查重功能会给出 AI 风格/人类风格倾向诊断，只作为参考，不直接改文稿。
- 写作模式保留模型选择器和右侧文学秘书，适合局部润色、材料整理和选区处理。

### 工作室 Agent

- 内置工作室 Agent 覆盖写作编辑、学术研究、内容运营、专业服务、产品支持和审核校验。
- “秘书长”不可禁用，负责主对话和调度；其他 Agent 可在设置中启用或禁用。
- 用户可以用自然语言生成自定义 Agent 草案，保存启用后参与调度。

## 工程稳定性原则

- 成本控制优先：简单任务不触发多 Agent，重复任务优先走缓存和摘要引用。
- 长上下文只在必要时传递；Agent 之间使用结构化输出和 outputId/摘要交接，减少重复上下文。
- 记忆默认本地，用户可编辑、禁用、删除。
- 所有新增运行时稳定性能力都在 Papyrus 本地实现，不引入新的通用 Agent 框架依赖。

## 版本

- 应用版本：0.1.2
- 主要变更：模型选择器修复、主站模型元数据显示、Hive 超时/重试/断路器/黑板、短期状态与长期记忆分层。
