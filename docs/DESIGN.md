# Papyrus 设计语言

## 产品定位

Papyrus 是面向真实写作工作的桌面工作台。设计服务于长时间阅读、修改、审校和多资料协作，不做营销页式装饰。

## 设计原则

- 内容优先：正文、资料和 Agent 输出必须比装饰更突出。
- 工具感清晰：按钮、状态、命令和模型选择都要可扫读。
- 低噪声：避免大面积渐变、过重阴影和无意义玻璃材质。
- 可恢复：生成、审阅、写入、拒绝、回退都要有明确状态。
- 密度适中：桌面端支持三栏工作，窄屏优先保留编辑与 Agent 输入。

## 视觉系统

- 主背景：接近纸张但不过度泛黄
- 主墨色：深黑褐，保证长文阅读对比
- 品牌强调：Scallion 绿色，主要用于当前状态、技能、资料和运行反馈
- 边界：薄边界优先，阴影只用于浮层和菜单
- 圆角：工具面板和菜单保持 8 到 12px，避免过圆

## 交互语言

- `/` 是命令入口
- `@skill` 是能力调度入口
- `#file` 是资料引用入口
- 文稿补丁必须经过 Auto 写入或人工审阅
- 用量浮球承担模型与上下文的轻量状态提示

## 动效

动效只服务于状态转场和注意力引导：

- 模式切换使用短距离 layout motion
- 菜单使用轻量出现动效
- 运行状态使用低强度进度反馈
- reduced motion 下内容必须直接可见
