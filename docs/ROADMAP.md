# Papyrus 路线图

## 竞品吸收

OpenWrite v1.2.0 的可借鉴点：

- 联网搜索入口
- `@skill` 技能提及
- `#file` 文件提及
- TXT 导出
- 用量悬浮球
- 主题背景分组
- 工具栏紧凑化
- 云同步和编辑器同步稳定性

Papyrus 已吸收并改造成更适合文学与文科写作的形态：技能不是简单快捷指令，而是进入 Agent 提示词和上下文的显式约束；文件提及不只是附件选择，而是进入项目上下文；用量浮球服务于长文工作台的上下文判断。

## 差异化方向

- 不只写网文，也支持说明文、议论文、记叙文、评论、散文、作业和知识问答。
- Companion 模式像文学秘书，能解释、搜索、诊断、改写和整理资料。
- Flow 模式像多 Agent 编辑部，能规划并执行复杂写作任务。
- Story System 保留长篇连载的伏笔、角色状态、章节提交和读者承诺。

## 下一步

- 更完整的文件检索和引用定位
- 章节库存与发布准备度可视化
- 主题分组：浅色、深色、专注、夜写
- 资源云同步和冲突解决
- 编辑器同步稳定性测试
- 更细的作文批改 rubric
- 面向文学常识的本地知识卡片
