# Papyrus 功能说明

## 工作台

Papyrus 的主界面由项目导航、编辑器、文学秘书面板、Flow 工作流和状态层组成。用户可以直接写作，也可以把任务交给 Agent。

## 文学秘书模式

文学秘书常驻编辑器右侧，适合低打断的即时协助。

- 解释文学常识、修辞、流派和写作概念
- 回答作文、作业和写作方法问题
- 处理选区改写、降噪、润色和终校
- 根据当前文稿做结构诊断
- 在需要资料时搜索并整理来源
- 通过 `@skill` 指定技能，通过 `#file` 引用资料

## Flow 模式

Flow 适合复杂任务和长结果。主笔 Agent 会先判断任务类型，再选择工具和子 Agent。

- 小说章节与连载生产
- 议论文提纲和论证链
- 说明文素材和结构
- 记叙文升格
- 文学常识问答
- 资料搜集和事实核验
- 文稿补丁写入或人工审阅

## `@skill`

可用技能包括：

- 文学正文起草
- 结构诊断室
- 检索与反证
- 文风连续性
- 终校清稿
- 档案与长期记忆
- 文科论证锻造
- 人物声纹
- 场景镜头调度
- 发表前编辑

显式输入 `@技能名` 会让 Agent 优先遵守该技能的工作规则。

## `#file`

导入的资料可以通过 `#文件名` 提及。Agent 会把对应文件摘要或正文片段加入上下文，用于引用、分析、改写和资料整理。

## 文体工作流

Papyrus 内置八类任务：

- 网文与长篇小说
- 记叙文
- 说明文
- 议论文
- 评论与批评
- 散文
- 文学常识与写作问答
- 资料搜集

每类任务都有不同的结构模板、素材提示和验收标准。

## 导出

- Word 导出：保留标题、段落和基础层级
- TXT 导出：UTF-8 编码，适合投稿、存档和跨软件迁移

## 用量浮球

工作台右下角显示上下文占用、文稿 tokens、对话 tokens、资料数量、当前模型和运行状态，帮助用户判断是否需要压缩上下文或减少资料。

## Remote Relay 远程通讯

Papyrus 可以通过 Scallion Relay 接收外部机器人消息，让手机端的微信、QQ、飞书和企业微信成为轻量入口。

- 微信/QQ 通过 Clawbot 或同类适配器接入，不直接操作私有协议。
- 飞书和企业微信通过官方机器人 webhook 接入。
- Papyrus 桌面端轮询远程任务，并可分流到文学秘书或 Flow 工作流。
- 处理完成后把结果回传给 Relay，再由对应机器人回复用户。
- 支持频道 ID、中继密钥、平台白名单、默认模式和轮询间隔配置。

详见 [Remote Relay 文档](REMOTE_RELAY.md)。
