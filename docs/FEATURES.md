# Papyrus 功能说明

## 工作台

Papyrus 的主界面由项目导航、编辑器、文学秘书面板、Flow 工作流和状态层组成。用户可以直接写作，也可以把任务交给 Agent。

## 电脑管家与工作助手

电脑管家模式让 Agent 在用户授权的工作区内完成可审阅的本机任务。它覆盖文件查找、资料检查和受限的批量整理，但不提供通用终端，也不会在没有确认的情况下写入磁盘。

### 可用能力

- **授权工作区**：用户先添加工作区或下载目录。Agent 只能使用根 ID 和根内相对路径。
- **本地检索**：扫描目录、搜索文件名或内容、读取文件摘要，并返回脱敏后的结果。
- **批量文件操作**：复制、移动、重命名、创建目录和移入 Papyrus 恢复库。冲突策略支持 `skip`、`rename` 和 `overwrite`。
- **桌面状态**：能力列表按当前平台返回可用状态。Windows、macOS 和 Linux 都使用原生适配器；不支持的平台不会回退到不安全的路径操作。

### 写入流程

批量写入共享预览、审批和取消机制，但不同操作只执行必要的原生步骤：

1. Agent 生成结构化计划，系统校验根 ID、相对路径、操作数量和冲突策略
2. 原生代理创建预览并读取源文件快照；移入恢复库、覆盖和跨设备移动等需要 recovery receipt 的事务还会预检同卷恢复库和 receipt 写入能力
3. 用户批准后，系统才消费审批令牌并执行；运行中可以取消
4. 执行结果分成已完成、已跳过、失败、待处理和警告，UI 保留可恢复项和重试入口

具体路径如下：创建目录只校验父目录并创建一个新叶子；复制到新目标会写入临时 staging 文件并以 no-replace 方式发布；新目标同卷移动或重命名直接使用源句柄和 no-replace rename；移入恢复库、覆盖和跨设备移动才会写入 recovery receipt。预览过期、文件身份改变、链接或重解析点出现、所需恢复库不可用，都会停止执行。系统不会自动改用绝对路径、普通覆盖式重命名或系统回收站来绕过安全检查。

### 恢复与回收站

移动、删除和覆盖不会直接丢弃文件。原生代理会在同一卷、授权父目录旁维护私有的 `.papyrus-recovery` 恢复库，并为每个对象生成 UUID 目录和 `receipt.json`。receipt 记录预览 ID、操作序号、原始相对路径和文件身份，用户可以据此追溯和恢复对象。

`file_trash` 的安全语义是移动到 Papyrus 恢复库。操作系统回收站不是安全原语；如果桌面适配器提供额外派发，它只能在 receipt 持久化并完成审计后 best-effort 执行。系统回收站失败时，Papyrus 仍以私有恢复库和 receipt 为准。

### 跨平台保护

Windows 会拒绝 reparse point 并比较 `FILE_ID_INFO`；Linux 使用 `openat`、`O_NOFOLLOW`、`fstatat(AT_SYMLINK_NOFOLLOW)` 和 `renameat2(RENAME_NOREPLACE)`；macOS 使用对应的 `openat` 组件检查和 `renameatx_np(RENAME_EXCL)`。三个平台都会在提交前重新验证身份、版本和内容摘要，并在发布前保留恢复路径。

能力列表按编译目标报告平台级支持：Windows、macOS 和 Linux 构建会静态列出原生文件能力，其他目标列为不可用。运行时仍会检查实际的 no-replace 原语、身份和恢复预检；如果旧内核缺少必需原语，执行会 fail closed，并在本次运行中禁用受影响操作。跨设备移动只有在复制、校验发布结果并成功回收原文件时才继续。

### UI 状态

- **已完成**：显示结果和可用的 `recoveryReceipts`
- **已取消**：显示取消原因和待处理条目，不伪造失败项
- **预览已过期**：提示重新生成预览，不改变未经确认的目标
- **部分事务**：明确说明已有对象进入恢复库，保留可恢复错误和调用方的 `previewId`；当前失败条目可能没有 `recoveryReceipts` 或新的 recovery 审计条目，UI 不应声称已经能按 `previewId` 自动定位 receipt，也不应声称没有发生变化
- **审计不可用**：保留已完成结果和 receipt，显示可恢复警告，不回滚文件
- **恢复清理警告**：提示空恢复槽仍需清理，保留主结果并允许重试

同一身份的恶意进程如果在 Papyrus 建立快照前已经持有破坏性 OS 句柄，非特权桌面应用无法完全撤销该句柄。这个操作系统边界会在能力说明中公开；它不允许模型控制路径，也不改变身份和快照校验规则。

## 文学秘书模式

文学秘书常驻编辑器右侧，适合低打断的即时协助。

- 解释文学常识、修辞、流派和写作概念
- 回答作文、作业和写作方法问题
- 处理选区改写、降噪、润色和终校
- 根据当前文稿做结构诊断
- 在需要资料时搜索并整理来源
- 通过 `@skill` 指定技能，通过 `#file` 引用资料

## Flow 模式

Flow 适合复杂任务和长结果。主笔 Agent 会先判断任务类型，再选择工具和子 Agent。

- 小说章节与连载生产
- 议论文提纲和论证链
- 说明文素材和结构
- 记叙文升格
- 文学常识问答
- 资料搜集和事实核验
- 文稿补丁写入或人工审阅

## `@skill`

可用技能包括：

- 文学正文起草
- 结构诊断室
- 检索与反证
- 文风连续性
- 终校清稿
- 档案与长期记忆
- 文科论证锻造
- 人物声纹
- 场景镜头调度
- 发表前编辑

显式输入 `@技能名` 会让 Agent 优先遵守该技能的工作规则。

## `#file`

导入的资料可以通过 `#文件名` 提及。Agent 会把对应文件摘要或正文片段加入上下文，用于引用、分析、改写和资料整理。

## 文体工作流

Papyrus 内置八类任务：

- 网文与长篇小说
- 记叙文
- 说明文
- 议论文
- 评论与批评
- 散文
- 文学常识与写作问答
- 资料搜集

每类任务都有不同的结构模板、素材提示和验收标准。

## 导出

- Word 导出：保留标题、段落和基础层级
- TXT 导出：UTF-8 编码，适合投稿、存档和跨软件迁移

## 用量浮球

工作台右下角显示上下文占用、文稿 tokens、对话 tokens、资料数量、当前模型和运行状态，帮助用户判断是否需要压缩上下文或减少资料。

## Remote Relay 远程通讯

Papyrus 可以通过 Scallion Relay 接收外部机器人消息，让手机端的微信、QQ、飞书和企业微信成为轻量入口。

- 微信/QQ 通过 Clawbot 或同类适配器接入，不直接操作私有协议。
- 飞书和企业微信通过官方机器人 webhook 接入。
- Papyrus 桌面端轮询远程任务，并可分流到文学秘书或 Flow 工作流。
- 处理完成后把结果回传给 Relay，再由对应机器人回复用户。
- 支持频道 ID、中继密钥、平台白名单、默认模式和轮询间隔配置。

详见 [Remote Relay 文档](REMOTE_RELAY.md)。
